# SKILL.md - 放样反馈模块 (v5.3.3)

## 核心定位

本模块接收放样人反馈的实际放样情况，解析稳定性研究计划，生成取样节点写入本地取样计划总表。

```
用户反馈 → 解析计划.xlsx → 计算取样日期 → 写入取样计划总表
```

---

## 命令行用法

```bash
# 处理该药品所有批号、所有条件
python3 skills/stability-feedback/scripts/feedback_v5.py \
  --drug "黄体酮注射液" \
  --placement-date "2026-04-09" \
  [--notes "无异常"]

# 只处理指定批号
python3 skills/stability-feedback/scripts/feedback_v5.py \
  --drug "黄体酮注射液" \
  --batch "LM-20260321-01,LM-20260321-02" \
  --placement-date "2026-04-09"

# 只处理指定条件
python3 skills/stability-feedback/scripts/feedback_v5.py \
  --drug "黄体酮注射液" \
  --condition "长期25℃" \
  --placement-date "2026-04-09"

# 仅预览（不写入）
python3 skills/stability-feedback/scripts/feedback_v5.py \
  --drug "黄体酮注射液" \
  --placement-date "2026-04-09" \
  --dry-run
```

## 解析逻辑（v5.3.1 改进）

### 时间点解析（v5.3.1）
- **天数字段**：从稳定性研究计划中取当前天数点与上一天数点的**间隔**（原始值差）
  作为 count_val，标签显示该间隔（如"10"）
  计划日期 = 放样日期 + 累计天数（月历点后第一个天数点从月历点起算，之后每个天数点从前一天数点起算）
- **月历列（1月/2月等）**：标签保留月历标签（如"1月"），
  计划日期 = 放样日期 + `relativedelta(months=月数)`
- **`距放样天数`**列：显示**实际距放样天数**（由计划日期反算）
- 两类点按**实际距放样天数**统一排序

### 检测项用量解析
- `✓` 或 `是` → **0**（检测但不消耗样品，不计入消耗量合计）
- `-` 或空 → **0**（不检测）
- 数字 → 该数字

### 计划文件读取
- 在 `stability_plans/{药品名}/` 下查找 `*_稳定性计划.xlsx`
- 读取「稳定性研究计划」工作表

## 取样计划总表

**路径**：`stability_plans/取样计划总表.xlsx`

**列顺序**（按 v5.0.0 调整，去掉备注列）：

| 列 | 类型 | 说明 |
|----|------|------|
| 项目名称 | 文本 | 药品名 |
| 批号 | 单选 | 批号 |
| 稳定条件 | 单选 | 长期25℃/阴凉20℃/加速40℃/高温60℃ |
| 实际放样日期 | 日期 | 放样基准日 |
| 时间点 | 文本 | 1月/2月/10天/20天... |
| 距放样天数 | 数字 | 实际天数（30/91/121...）|
| 计划取样日期 | 日期 | 自动计算（基于实际放样日期） |
| 实际取样日期 | 日期 | 取样完成后填入 |
| 状态 | 单选 | 待执行/已完成/逾期 |
| 取样量 | 数字 | 本次总消耗（支）|
| 外观/性状 | 数字/✓ | ✓=检测但不消耗，数字=消耗量 |
| 有关物质 | 数字 | 本次用量（支）|
| pH值 | 数字 | 本次用量（支）|
| 不溶性微粒 | 数字 | 本次用量（支）|
| 含量 | 数字 | 本次用量（支）|
| 无菌 | 数字 | 本次用量（支）|
| 细菌内毒素 | 数字 | 本次用量（支）|

**日期格式**：Excel 中以 `YYYY-MM-DD` 显示，不再是时间戳数字。
