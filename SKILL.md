# SKILL.md - 放样反馈模块 (v6.1.0)

## 核心定位

本模块接收放样人反馈的实际放样情况，通过三步流程生成取样节点写入本地取样计划总表。

```
用户反馈 → 自动扫描计划文件 → Agent 分类判断 → 脚本计算写入
```

**关键设计**：不写死任何条件的计算方式，由 Agent 根据扫描报告自主判断。

---

## 三步执行流程

### 步骤1：扫描（Agent 执行）

```bash
python3 skills/stability-feedback/scripts/feedback_v6.py \
  --drug "黄体酮预灌封注射液" \
  --placement-date "2026-04-22" \
  --scan
```

脚本自动扫描计划文件，输出诊断报告，内容包括：
- 药品名、批号列表
- 每个条件区块：条件名、表头行/时间点行/月历行行号、检测项、天数列、月历列

Agent 读取报告内容，进行分类判断。

### 步骤2：Agent 分类（Agent 判断）

Agent 阅读扫描报告后，进行**一致性检查**，再决定计算方式：

**① 批号一致性检查（关键）**
Agent 将用户反馈中提到的批号与扫描报告中计划文件的批号进行比对：
- **部分匹配**：用户只提到部分批号 → **发出警告提示**，询问"计划文件中还有其它批号，是否一起处理？"
- **完全匹配**：用户提到的批号全部包含在计划文件中 → 继续
- **超出计划文件**：用户提到的批号在计划文件中找不到 → **发出错误提示**，终止流程

**② 多计划表歧义检查**
在扫描阶段若发现多个计划文件包含相同批号：
- **完全匹配**：多个计划表批号集合相同 → **主动发出警告**，提示"批号在多个计划表中均有出现，请确认使用哪个"
- **优先选择匹配度最高的**，但需用户确认后再执行

**③ 计算方式判断（完成①/②后）**

Agent 为每个条件区块决定计算方式：


**`month`（月历型）**：按日历月计算，适合长期/阴凉/加速等正式稳定性研究。
> 特征：时间点为"1月""2月"等月历标签，用 `relativedelta(months=N)` 计算。

**`day`（天数型）**：
> 按天数间隔累加，适合高温/高湿/光照/中间条件等。
> 特征：时间点为"5天""10天"等纯天数值，直接加天数。

**Agent 输出格式**：
```json
{"阴凉20℃": "month", "高温60℃": "day", "中间条件": "month"}
```

⚠️ **必须先完成①和②的检查，再输出上述 JSON**。

### 步骤3：执行（脚本执行）

```bash
python3 skills/stability-feedback/scripts/feedback_v6.py \
  --drug "黄体酮预灌封注射液" \
  --placement-date "2026-04-22" \
  --classify '{"阴凉20℃": "month", "高温60℃": "day"}'
```

脚本根据 Agent 判断，对每个条件应用对应逻辑，计算取样日期，写入取样计划总表。

## 完整流程（一步执行）

```bash
# 1. 先扫描
python3 skills/stability-feedback/scripts/feedback_v6.py \
  --drug "黄体酮预灌封注射液" \
  --placement-date "2026-04-22" \
  --scan

# 2. Agent 读取报告，判断后传入 --classify
python3 skills/stability-feedback/scripts/feedback_v6.py \
  --drug "黄体酮预灌封注射液" \
  --placement-date "2026-04-22" \
  --classify '{"阴凉20℃": "month", "高温60℃": "day"}'

# 仅预览（不写入）
python3 skills/stability-feedback/scripts/feedback_v6.py \
  --drug "黄体酮预灌封注射液" \
  --placement-date "2026-04-22" \
  --classify '{"阴凉20℃": "month", "高温60℃": "day"}' \
  --dry-run
```

## 检测项用量解析

- `✓` 或 `是` → **0**（检测但不消耗样品，不计入消耗量合计）
- `-` 或空 → **0**（不检测）
- 数字 → 该数字

## 取样计划总表

**路径**：`stability_plans/取样计划总表.xlsx`

**列顺序**：

| 列 | 类型 | 说明 |
|----|------|------|
| 项目名称 | 文本 | 药品名 |
| 批号 | 单选 | 批号 |
| 稳定条件 | 单选 | 条件名（由扫描自动发现）|
| 实际放样日期 | 日期 | 放样基准日 |
| 时间点 | 文本 | 1月/2月/10天/20天... |
| 距放样天数 | 数字 | 实际天数 |
| 计划取样日期 | 日期 | 自动计算（基于实际放样日期） |
| 实际取样日期 | 日期 | 取样完成后填入 |
| 状态 | 单选 | 待执行/已完成/逾期 |
| 取样量 | 数字 | 本次总消耗（支）|
| 外观/性状 | 数字/✓ | ✓=检测但不消耗 |
| 有关物质 | 数字 | 本次用量（支）|
| pH值 | 数字 | 本次用量（支）|
| 不溶性微粒 | 数字 | 本次用量（支）|
| 含量 | 数字 | 本次用量（支）|
| 无菌 | 数字 | 本次用量（支）|
| 细菌内毒素 | 数字 | 本次用量（支）|

**日期格式**：Excel 中以 `YYYY-MM-DD` 显示。
